### 2.2 对 LARP 既有代码的三处修改（唯一动老代码的地方）

**(a) torchrun 入口**（`train.py`）：删 `mp.spawn(main_worker, nprocs=tot_gpus)`；改为读取 `LOCAL_RANK/WORLD_SIZE/RANK` 环境变量，`init_process_group("nccl")` 直接初始化；启动命令统一 `torchrun --nproc_per_node=8 --nnodes=N train.py --cfg ...`。理由：mp.spawn 是单机风格，多机与容器调度不友好。

**(b) 梯度累积**（`trainers/base_trainer.py` 训练循环处）：包一层 `for micro in range(grad_accum): loss = loss / grad_accum; scaler.scale(loss).backward()`，累满才 `optimizer.step()`；DDP 下前 k−1 个 micro-step 用 `model.no_sync()` 省通信。判别器侧同构。配置新增 `grad_accumulates: int`。理由：Stage 1 全局 batch 256，8 卡下每卡 32 会 OOM（视频 17 帧），需累积。

**(c) 三阶段状态机**（新文件 `uvt_tokenizer_trainer.py`，继承 base_trainer）：
- `stage: 1|2|3` 由 config 决定；构造期按 stage 设置 `requires_grad`（S2 冻 encoder+GSB，S3 冻 encoder+decoder 只留 Sem-ViT/池化头）与损失开关；
- S3 启动时先跑一遍 `estimate_latent_stats()`：在 10k 样本上前向统计 z 的逐通道 mean/std 写入 GSB buffer 并置 `normalize=True`，随 checkpoint 保存；
- S2 的 G/D 交替沿用 LARP 的实现（d_update_freq 等旋钮原样）。

### 2.3 联合 loader 规格（`datasets/joint_loader.py`，新写）

- 输入：`sources: [{dataset, batch_size, ratio}]`（例：图像 bs=32 ratio=1、视频 bs=8 ratio=3 → 每 4 个 step 3 视频 1 图像）；
- 每个 step 产出**单源纯 batch**（不混装，因视频/图像张量形状不同、tubelet mask 不同）；
- **确定性**：采样序列由 `(seed, epoch)` 完全决定并落盘 `sampling_trace.json`——配对消融（D13）依赖同数据序；
- 多分辨率（P4 起）：按 OmniTokenizer 的 `resolution_scale` 思路做分辨率 bucket，同 batch 同分辨率。

### 2.4 SigLIP2 加载细则（`siglip_backbone.py`）

```python
from transformers import SiglipVisionModel  # siglip2 权重兼容此类加载
def load_split_backbone(name="google/siglip2-so400m-patch16-naflex", gen_depth=13):
    vt = SiglipVisionModel.from_pretrained(name).vision_model
    assert vt.config.num_hidden_layers == 27 and vt.config.hidden_size == 1152
    blocks = vt.encoder.layers
    return dict(
        patch_embed = vt.embeddings,           # 空间patchify+学习式PE（ADR-8保留）
        gen_blocks  = blocks[:gen_depth],      # 1–13
        sem_blocks  = blocks[gen_depth:],      # 14–27
        map_head    = vt.head,                 # MAP池化头 → 复制两份:一份进Sem-ViT(可训),一份留教师(冻结)
    )
```
- decoder 27 层 = `deepcopy(blocks)` 全部 + 随机像素头；
- 教师 = 同权重独立实例 `.eval().requires_grad_(False)`，预处理必须走官方 `AutoProcessor`（golden test 锁定，见 §5）。

## 3. Phase B 实验仓（fork `FoundationVision/OmniTokenizer`，一次性改造）

> 全部改动 ≤200 行，文件:行号基于 2026-07-07 主分支。改完打 tag `phase-b-v1`，实验结束归档。

### 3.1 改造 1：时间注意力三模式（~40 行）

- `OmniTokenizer/modules/attention.py`：`Attention.__init__` 增参 `temporal_attn_mode: str = "causal"`；`forward` 中（SDPA 路径 :451）：

```python
if self.temporal_attn_mode == "tubelet":
    T1 = N  # 此处序列即时间轴((b h w) t d)，N=时间位数
    idx = torch.arange(T1, device=q.device)
    allowed = (idx[None] == idx[:, None]) | (idx[None] == idx[:, None] - 1)
    out = F.scaled_dot_product_attention(q, k, v, attn_mask=allowed, scale=self.scale)
elif self.temporal_attn_mode == "causal":
    out = F.scaled_dot_product_attention(q, k, v, is_causal=True, scale=self.scale)
else:  # full
    out = F.scaled_dot_product_attention(q, k, v, scale=self.scale)
```

- 下传链路：`omnitokenizer.py:857-861`（时间 transformer 构造）→ `Transformer(...)` → `Attention(...)` 增 kwargs；`vqgan_train.py` 加 `--temporal_attn_mode` 参数。
- **纪律**：三个模式必须都走 SDPA 路径（官方 README 承认 SDPA 与手写路径数值不等价，混用即引入实现混杂）。

### 3.2 改造 2：层级时间 patchify 三臂（~80 行）

- 臂 1（单步 4×，基线）：原样，`--temporal_patch_size 4`；
- 臂 2（学习式 2×+2×）：`omnitokenizer.py` `to_patch_emb`（:814-822）改 `pt=2`；在 `OmniTokenizer_Encoder.encode()` 空间与时间 transformer 之间（:894-903 间）插入 `TemporalFold2x`（从主仓 `temporal_fold.py` 拷贝，`Linear(1024→512)`）；decoder 对称：`decode()`（:1071-1084 间）插 `TemporalUnfold2x`，`to_pixels`（:1012-1017）改 `pt=2`。注意锚帧在此仓天然隔离（:910-914 pool 只作用非锚帧），Fold 同样只作用 `tokens[:,1:]`；
- 臂 3（2×+AvgPool2×，无参层级）：`--temporal_patch_size 2 --defer_temporal_pool`（:792-795 现成机制，零改动）。
- 新增 `--temporal_fold_mode {single,learned,avgpool}` 统一开关。
