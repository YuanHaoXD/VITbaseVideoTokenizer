## §1 项目是什么（30 秒认知）

**UVT（Unified Video Tokenizer）**：用**单个 ViT**（SigLIP2-So400M 初始化）做一个**统一视频 tokenizer**，同时满足三件事：

1. **图像+视频统一**：一套权重处理两种输入（1 锚帧 + 16 帧 clip 协议）
2. **重建+语义统一**：latent 既能高保真重建像素（对标 Wan2.2），又携带可线性读出的语义（对标 AToken）
3. **latent 可生成性**：冻结 tokenizer 后在 latent 上训 DiT 收敛良好

**不做什么**（重要）：不做 UMM（不接 LLM 做端到端系统）；不做离散主线（FSQ 仅附赠）；不做长视频生成系统。

**理论来源**：借鉴 Hydra-X（arXiv:2606.13289）的 tokenizer 设计，**收窄为"只做 tokenizer"**——论文知识 10% 成本，系统工程 90%；砍掉 UMM 的数据工程与论文含糊点。

**交付物**：encoder/decoder 权重（HF 可加载）+ 训练代码 + 论文级实验矩阵。

## §2 为什么这么干（核心赌注 + 科学贡献）

### 2.1 三个核心设计（借鉴 Hydra-X，均有消融证据）

1. **tubelet 因果注意力**（"少即是多"）：时间位 t 只注意 {t−1, t}。Hydra-X 证明全时空注意力**破坏**图像预训练的局部性先验 → 最小时间感受野反而最优（最快 0.17s + 最好 PSNR）。
2. **层级时间 patchify**：两个 2× 折叠阶段替代单步 4×（渐进多尺度折叠时间轴），优于单步。
3. **Decompressor 双教师蒸馏**：解决"视频 latent 被压到 1+T/4 位、而视频教师在原帧率"的不对称——轻量 4× 时间上采样器（训完丢弃）把压缩 latent 升回原帧率接受视频教师蒸馏。

### 2.2 我们的科学增量（论文卖点之一）

Hydra-X 的"少即是多"结论**可能依赖图像预训练先验**。我们做 **Phase B 边界条件研究**：在**从头训练的 OmniTokenizer**（无预训练先验）上重做 tubelet/层级 patchify 消融，看结论是否保持/消失/翻转。**这是 Hydra-X 未做的独立知识贡献。**

### 2.3 SigLIP2 一模三用（核心卖点的前提）

`google/siglip2-so400m-patch16-256` 同时是：① Gen/Sem-ViT 与 decoder 的**初始化**（"浅层=结构、深层=语义"的层级先验）② 冻结**图像教师**（蒸馏）③ **文本塔**（zero-shot 评测）。所以这个权重**必须下载**，是 P1-smoke 硬前提。

### 2.4 关键决策（D1–D15 摘要，详见 README §2）

- **D1** 只做 tokenizer 不做 UMM　**D2** 连续 latent 主线（FSQ 附赠）　**D3** 自建标准 ViT（SigLIP2 初始化，不基于现成 tokenizer 架构）　**D4** 工程骨架 = LARP base_trainer 改造（唯一无 Lightning）　**D5** GAN = LARP transformer 判别器+LeCam　**D6** Phase B 在 OmniTokenizer 上做　**D7** 锚帧隔离（主动偏离 Hydra-X 的 zero-pad）　**D8** 教师 = SigLIP2+InternVideo-Next-L　**D9** 五锚点校准（主判据=排序复现）　**D10** 数据=ImageNet+UCF+OpenVid（禁 WebVid）　**D11** 语义评测含 CKNNA（禁对教师自身）　**D12** 可生成性=冻结 tokenizer+DiT　**D13** 消融≥2 seed 配对比较　**D14** KL 权重 1e-6 起扫　**D15** 无 GAN 变体作 Stage-2 平行对照

## §3 架构心智模型（张量流 + 部件 + 为什么）

### 3.1 总体张量流（Stage 1，256²，1 锚帧 + 16 帧，D=1152）

```
x: 图像[B,3,256,256] / 视频[B,3,17,256,256]
  ①空间 patchify 16×16（继承 SigLIP2）②时间折叠阶段一 2×（仅非锚帧；锚帧隔离）
  → tokens [B,1+8,256,1152]
  ③Gen-ViT 前段（SigLIP2 第1–6层，tubelet mask）④时间折叠阶段二 2× → [B,1+4,256,1152]
  ⑤Gen-ViT 后段（第7–13层）→ h [B,1+4,256,1152]
  ⑥GSB 压缩 Linear(1152→128)→(μ,ρ)→重参数化 → z [B,1+4,256,64]  ← 唯一对外 latent
  ├─► ⑦Decoder（27层对称 ViT + 两个 TemporalUnfold2x）→ x̂ [B,3,17,256,256]
  │     损失：L1 + LPIPS + KL（+Stage2 GAN）
  └─► ⑧GSB 反投影（在各消费方：sem_vit.in_proj / decoder.in_proj）
       ⑨Sem-ViT（第14–27层，tubelet mask）+ MAP 池化头 → s + s_pool
       ├─ s[:,0]+s_pool ──cosine──► SigLIP2 教师（patch+池化）
       └─ s[:,1:] ──► ⑩Decompressor(4×时间上采样,训完丢弃) ──► InternVideo 教师
```

**压缩率**：空间 16×、时间 4×、通道 1152→64。与 Hydra-X 完全一致。

### 3.2 各部件 + 为什么（详见 01 §2）

| 部件                                     | 为什么                                                                             |
| -------------------------------------- | ------------------------------------------------------------------------------- |
| Gen-ViT / Sem-ViT（SigLIP2 从 13/14 层切开） | HYDRA 消融：均衡切分最优（13+14）                                                          |
| tubelet mask                           | "少即是多"（见 §2.1）                                                                  |
| 层级 2×+2× 折叠                            | 渐进多尺度 > 单步 4×                                                                   |
| 锚帧隔离（ADR-4'）                           | **主动偏离** Hydra-X 的 zero-pad——行业标准（OmniTokenizer/Wan-VAE/LeanVAF 同款），消掉 pad 位置歧义 |
| GSB C=64                               | HYDRA 消融：C≥256 生成崩坏，64 是三方甜点                                                    |
| Decompressor                           | 视频语义监督的不对称解法（见 §2.1）                                                            |
| Decoder 用 SigLIP2 权重复制初始化              | HYDRA：decoder 越大越好；结构先验对重建有益                                                    |

### 3.3 ADR 裁决记录（主动偏离/未决项，详见 01 §3）

- **ADR-2** tubelet mask 以折叠后时间位为单位（高置信）
- **ADR-3** 折叠位置 {(0,6)} 默认，P2 预消融后冻结（**低置信——最大遗留歧义**）
- **ADR-4'** 锚帧隔离替代 zero-pad（P2 保留对照臂）
- **ADR-5** Stage-3 规范化=定义规范 latent 接口 ẑ；decoder 物理消费反归一化 z
- **ADR-8** 位置编码=保留 SigLIP2 空间学习式 PE + 时间 RoPE（P1 消融）
- **ADR-9** 蒸馏含池化项（MAP head 对齐），换 zero-shot 文本塔

**性质与教训**：(a) 论文公式→规格转录会丢"管道细节"（论文从不写 LN）；(b) tiny 随机初始化没有巨激活，84 测试对尺度病天然免疫——**tiny 冒烟和真训练之间缺"真权重前向统计探针"这一层**（激活幅值/μ 分布/输出值域，单卡几分钟），以后凡切开预训练网络造新接口，切口两侧默认过 LN 并跑一次探针。

我给 base_trainer 加了 8 处守卫（device/GradScaler/数据搬运/DDP device_ids/autocast/rng×2/empty_cache），让代码在无 CUDA 时也能跑（开发机验证用）。**GPU 路径零改动**（is_available=True 时走原路径），3 个 LARP 同源 trainer 不受影响。你看到这些守卫不要误判为问题。
