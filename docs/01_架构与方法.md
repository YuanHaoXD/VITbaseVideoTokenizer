## 1. 总体张量流（一图定全局）

以 Stage 1 训练分辨率 256²、clip = 1 锚帧 + 16 帧为例。patch=16 → 每帧 16×16=256 个空间 token；D=1152（SigLIP2-So400M 宽度）。

```
输入 x: 图像 [B,3,256,256] 或视频 [B,3,17,256,256]
│
│ ①空间 patchify 16×16（SigLIP2 patch embed 权重继承）
│ ②时间 patchify 阶段一（2×，仅非锚帧；锚帧隔离，见 ADR-4'）
▼
tokens [B, 1+8, 256, 1152]（图像时 [B,1,256,1152]）
│
│ ③Gen-ViT 前段：SigLIP2 第 1–6 层（tubelet 因果注意力 + 位置编码见 ADR-8）
│ ④时间 patchify 阶段二（2×）→ [B, 1+4, 256, 1152]
│ ⑤Gen-ViT 后段：SigLIP2 第 7–13 层
▼
h [B, 1+4, 256, 1152]
│
│ ⑥GSB 压缩：Linear(1152→128) → (μ,ρ) → z = μ + ε·exp(0.5ρ)
▼
z [B, 1+4, 256, 64]   ←—— 唯一对外 latent（生成/重建/语义的共同接口）
│                          Stage 3 起经 ẑ=(z−m)/s 规范化（ADR-5）
├───────────────► ⑦Decoder（27 层 ViT，SigLIP2 权重复制初始化 + 随机 pixel head）
│                   含两个 TemporalUnfold2x（镜像④②的位置）→ x̂ [B,3,17,256,256]
│                   损失：L1 + LPIPS + KL (+Stage2 GAN)
│
│ ⑧GSB 反投影：Linear(64→1152)
▼
│ ⑨Sem-ViT：SigLIP2 第 14–27 层（tubelet 因果注意力）
▼
s [B, 1+4, 256, 1152] + 池化 token s_pool
├─ s[:,0]（锚帧）+ s_pool ──cosine──► SigLIP2 教师（patch 特征 + 池化嵌入，ADR-9）
└─ s[:,1:] ──► ⑩Decompressor（4×时间上采样，训完丢弃）──cosine──► InternVideo-Next 教师
```

**压缩率**：空间 16×、时间 4×（16 帧→4 个时间位 + 1 锚位）、通道 1152→64。与 Hydra-X 完全一致（z ∈ R^{64×(1+T/4)×H/16×W/16}）。

## 2. 各部件规格与设计理由

### 2.1 骨干：SigLIP2-So400M 视觉塔（为什么是它）

- 27 层、宽 1152、16 头、patch16 的标准 ViT（加载时断言 `config.num_hidden_layers==27`，权重 `google/siglip2-so400m-patch16-naflex`，Apache-2.0）。
- **一模三用**：① Gen/Sem-ViT 与 decoder 的初始化（"浅层=结构、深层=语义"的层级先验是切分方案的前提，随机初始化没有这个性质）；② 冻结副本做图像教师；③ 文本塔在评测期做 zero-shot 分类。
- 切分点 13/14：HYDRA 消融证明均衡切分最优（24 层上 12+12 完胜 16+8 和 24+0），27 层取最接近的 13+14；做成 config 字段 `gen_depth: 13`。

### 2.2 时间 patchify（层级 2×+2×，为什么分两步）

- Hydra-X 表 1：同压缩率下层级式全面优于单步 4×（DAVIS rFVD 11.19 vs 13.69）；机理假设是时间轴受益于渐进多尺度折叠。
- 折叠算子 = 相邻 2 时间位在通道维拼接 + `Linear(2D→D)`（等价 Conv3d(kernel_t=2,stride_t=2)），逆算子 = `Linear(D→2D)` + 通道→时间重排。
- **插入位置**（ADR-3，全案最大遗留歧义）：默认 {输入处, Gen-ViT 第 6 层后}；P2 预消融网格 {(0,6),(0,4),(4,9)} 后冻结。为什么放 Gen-ViT 内而不进 Sem-ViT：折叠必须发生在 GSB 之前（latent 时间长度由此决定）。

### 2.3 锚帧协议（ADR-4'：主动偏离原论文的地方）

- Hydra-X 原文：锚帧 zero-pad 后与其余帧过相同折叠算子。**我们改用锚帧隔离**（锚帧独立 patchify/独立像素头，永不折叠）——OmniTokenizer/Wan-VAE/LeanVAE 三家实读确认这是行业标准，且消除"pad 在前还是后"的歧义。
- 代价与保险：这是对论文的主动偏离，P2 保留一个 zero-pad 对照臂（1 组配对实验）验证两者无显著差异后，在论文里如实声明。
- 图像输入 = 只有锚帧：天然统一，无需任何特判逻辑之外的处理。

### 2.4 注意力与 mask（tubelet，为什么"少即是多"）

- Gen-ViT 与 Sem-ViT 全部使用**联合时空注意力 + tubelet 因果 mask**：时间位 t 的 token 可注意 {t−1,t} 两个时间位的全部空间 token；锚位只见自己。帧内全双向。
- 机理（论文解释+我们的边界条件假设）：全局时空注意力破坏图像预训练学到的局部性先验——因此该结论**可能依赖预训练初始化**，Phase B 在从头训练的 OmniTokenizer 上验证边界（见 04）。
- Sem-ViT 也用 tubelet 而非双向：Hydra-X 表 2 最后一行，双向使全部指标退化。
- **实现选型**：序列长度小（(1+4)×256=1280；阶段一期间 (1+8)×256=2304），直接构造稠密 bool mask 传 SDPA `attn_mask` 即可（2304² bool ≈ 5MB，可按形状缓存）；flex_attention 作为可选优化而非依赖（规避其 OOM 与 Windows 兼容风险）。每次 patchify 后时间位数变化，mask 需按当前 (T1, N) 重建（lru_cache）。

```python
# uvt/models/attention_mask.py
import torch
from functools import lru_cache

@lru_cache(maxsize=64)
def tubelet_mask(T1: int, N: int, kind: str, device_str: str) -> torch.Tensor:
    """返回 [S,S] bool（True=允许注意），S=T1*N。kind∈{full,causal,tubelet}。
    时间位 0 为锚位。tubelet: t 可见 {t-1, t}；锚位只见自己。"""
    device = torch.device(device_str)
    t_ids = torch.arange(T1, device=device).repeat_interleave(N)  # [S]
    tq, tk = t_ids[:, None], t_ids[None, :]
    if kind == "full":    return torch.ones(T1*N, T1*N, dtype=torch.bool, device=device)
    if kind == "causal":  return tk <= tq
    if kind == "tubelet": return (tk == tq) | (tk == tq - 1)
    raise ValueError(kind)
```

### 2.5 位置编码（ADR-8）

- 默认：**保留 SigLIP2 的学习式空间位置嵌入**（256² 下 16×16 网格与预训练一致，高分辨率时双三次插值）+ **时间轴加 RoPE**（新增，随机初始化无冲突）。理由：最大限度保护预训练先验；AToken 的先例证明 SigLIP 初始化 + RoPE 增补可行。
- 消融臂：全 3D RoPE（Hydra-X 声明的做法）。P1 各训 50k 迭代比重建，取优冻结。置信度：中。

### 2.6 GSB（连续瓶颈，为什么 64 维）

- HYDRA 消融：C≥256 时生成崩坏，C=64 是"重建-理解-生成"三方甜点。
- 结构（HYDRA 原文式 2）：`proj: Linear(1152→128)` → chunk 成 (μ,ρ) → 重参数化；`unproj: Linear(64→1152)` 进 Sem-ViT/decoder。KL = −½Σ(1+ρ−μ²−e^ρ)，权重 1e-6 起扫 {1e-6,1e-5,1e-4}（D14）。
- 一致性损失 L_cos（HYDRA 有、Hydra-X 未提）：unproj(μ) 与瓶颈前特征 h 的方向对齐，λ=1.0。实现为开关 `use_cos_consistency`，默认开，P1 消融验证。
- **修订 2026-07-12（第 15 号修复，docs/06 §6.8 / docs/08 §6.5）**：`compress` 入口**必须先过 `nn.LayerNorm(d_model)` 再 proj**——h 是 SigLIP2 残差流中段裸切片，含 O(10³) 巨激活通道（近图像无关），裸接 Linear 会导致 μ 跨图余弦 ~1（塌缩假象）+ ρ 钉死 clamp 上界（σ=e^10 噪声 + clamp 外零梯度死区），P1-smoke overfit 原理上不可收敛（NPU 实训抓出）。下方参考代码为原始版（含已删的 expand，见 08 §7），**以 `uvt/models/uvt/gsb.py` 实际代码为准**。同时 L_cos 的对齐目标 h 改为规范化后的 `gsb.norm(h)`（裸 h 的方向对齐会被共享巨激活平凡满足）。

```python
# uvt/models/gsb.py
class GSB(nn.Module):
    def __init__(self, d_model=1152, c_latent=64):
        super().__init__()
        self.proj   = nn.Linear(d_model, 2 * c_latent)
        self.unproj = nn.Linear(c_latent, d_model)
        self.register_buffer("z_mean", torch.zeros(c_latent))
        self.register_buffer("z_std",  torch.ones(c_latent))
        self.normalize = False                      # Stage 3 置 True（ADR-5）
    def compress(self, h):                          # h:[B,T1,N,D]
        mu, rho = self.proj(h).chunk(2, dim=-1)
        rho = rho.clamp(-30.0, 20.0)                # LeanVAE 同款数值保护
        z = mu + torch.randn_like(mu) * torch.exp(0.5 * rho)
        kl = -0.5 * (1 + rho - mu.pow(2) - rho.exp()).mean()
        return z, mu, kl
    def to_canonical(self, z):   return (z - self.z_mean) / self.z_std if self.normalize else z
    def from_canonical(self, zc): return zc * self.z_std + self.z_mean if self.normalize else zc
    def expand(self, z_canonical): return self.unproj(z_canonical)     # Sem-ViT/decoder 入口
```

### 2.7 Decoder（27 层对称 ViT）

- 与 encoder 同构：`unproj → 27 层 ViT block（tubelet mask）→ 两个 TemporalUnfold2x（位置镜像 encoder 折叠点）→ 像素头`。
- 锚帧与非锚帧各自独立像素头（`Linear(D→3·16·16)` / `Linear(D→3·pt·16·16)` 后 rearrange），镜像 OmniTokenizer 的做法。
- 初始化：SigLIP2 27 层权重整体复制 + 像素头随机。理由：HYDRA 证明 decoder 越大越好（358M>144M 单调），而"结构先验"对重建同样有益；这是推断而非论文陈述，P1 加"随机初始化 decoder"对照臂（一次性 50k 迭代）。
- **修订 2026-07-12（第 15 号修复）**：①末 block 之后、像素头之前**必须有 `final_ln: nn.LayerNorm(dim)`**——SigLIP2 原序是 encoder→post_layernorm→head（Sem-ViT 已镜像，decoder 此前漏了），否则真权重残差流（尺度 O(10²)）直入随机头产出 ±44 的 x_hat（目标 [0,1]）；②像素头校准初始化 weight std=0.02 / bias=0.5（初始输出≈灰图）；③另：encoder 侧 patchify 前须按 processor 统计做 SigLIP 输入归一化（GenViT `px_mean/px_std` buffer），入口协议 [0,1] 不变。以实际代码为准。

### 2.8 Decompressor 与双教师蒸馏（ADR-7/9）

- Decompressor：`2 × (TemporalUnfold2x → 双向 transformer block)` + `Linear(1152→D_teacher)`，把 s[:,1:] 从 4 个时间位升回 16 帧。**只在训练图上存在，checkpoint 不导出**。
- 蒸馏损失（式 3 的忠实实现 + 池化项扩展）：

```
L_dist = d_cos(s_anchor_patch, T_img_patch(x0))          # 锚帧 patch 级
       + d_cos(s_pool, T_img_pool(x0))                   # 池化嵌入级（ADR-9，zero-shot 的关键）
       + d_cos(Decompressor(s[:,1:]), T_vid(x_1:16))     # 视频级（纯图像 batch 屏蔽此项）
```

- **ADR-9 为什么要池化项**：SigLIP2 的图文对齐住在注意力池化后的图像嵌入里，不在原始 patch 特征里。只蒸 patch 特征无法继承文本塔；加一个 MAP 池化 token 对齐其池化嵌入，zero-shot 评测才成立。s_pool 由 Sem-ViT 末端一个注意力池化头产生（结构照抄 SigLIP2 的 MAP head 并用其权重初始化）。
- **ADR-7 视频教师对齐**：InternVideo-Next-L 输出 [B,16,N_t,D_t]，与 Decompressor 输出在空间网格双线性对齐后逐 token cosine；教师输入协议（帧采样/归一化）以官方 processor 为准并锁 golden fixture。备胎 InternVideo2-Stage2-1B（教师更换只影响绝对值不影响消融趋势）。
- λ_dist 默认 0.5，P3 扫 {0.25, 0.5, 1.0}（以重建-探针 Pareto 取舍）。

### 2.9 损失总表（分阶段）

| 阶段 | 激活损失（权重） | 训练参数 | 步数 |
|---|---|---|---|
| Stage 1 基础 | L1(1.0)+LPIPS(1.0，扫{0.5,1,4})+KL(1e-6 起扫)+L_dist(0.5)+L_cos(1.0 可关) | 全部 | 300k |
| Stage 2 精修 | 上表 − L_dist − L_cos + GAN（LARP 配方：transformer-D 384×8L，ns_smooth，LeCam 1e-3，disc_weight 0.3，D lr=G lr×1，Adam(0.5,0.9)，d_update_freq 5）；平行臂：Gram loss / DINOv3 感知损失替代 GAN | 仅 decoder（encoder/GSB 冻结） | 100k |
| Stage 3 调和 | 同 Stage 1 的蒸馏项 | 仅 Sem-ViT（+池化头）；先统计 z 通道 mean/std 并固化进 buffer | 50k |

优化器（G 侧）：AdamW(0.9,0.95) wd 0.05，峰值 lr 2e-4（Hydra-X 附录 A.2），warmup 5k，cosine 到 1e-5；bf16；EMA decay 0.9999（用 LARP 的多 decay EMA 机制同时存 0.999 备查）。

### 2.10 表征级 STI（Phase 5 的第二实验）

```python
# 编辑对 (x_src, x_tgt) 的两种路由（唯一差异变量）
def encode_pair_sti(tok, x_src, x_tgt):
    z_s = tok.encode_image(x_src)                    # Gen-ViT 独立编码（无跨帧注意力）
    z_t = tok.encode_image(x_tgt)
    pair = torch.cat([z_s, z_t], dim=1)              # 当长度2 clip: [B,2,N,64]
    s = tok.sem_vit(tok.gsb.expand(tok.gsb.to_canonical(pair)), mask="tubelet")
    return s[:, 0], s[:, 1]                          # s_src 只见 z_s; s_tgt 见两者
# 对照 encode_pair_indep: 两图分别单独过 sem_vit，其余全同
```

- 验证内容（无 LLM 的表征级 claim）：① 联合编码不损害源图 latent 的重建（decoder(z_s) 的 PSNR 两组应相同——z 未变，作 sanity）；② s_tgt 对源图结构的线性可读性（用 s_tgt 预测源图的低层属性/重建源图 patch 的探针）显著高于 Indep 组。协议细节在 04 §5。
