## 2. 数据处理管线（负责人：熟 LeanVAE 同学）

### 2.1 视频转码与切片（一次性离线处理）

```bash
# OpenVid 下载后统一转码: 短边256(Stage1用)与512(P4用)两档, h264 crf18, 保留原fps
ffmpeg -i in.mp4 -vf "scale=-2:256" -c:v libx264 -crf 18 -an out_256.mp4
```
- 转码理由：训练期 decord 随机解码 1080p 原片是 IO/CPU 灾难；256 档全量约 1–1.5 TB，可接受；
- **clip 采样规则（训练）**：每次 `__getitem__` 随机起点取**连续 17 帧**（不跳帧，fps 抖动 ±25% 做增广）；视频不足 17 帧的丢弃（预筛）；
- 元数据落盘为 LARP 式 CSV（`path,num_frames,fps,height,width`），沿用主仓 `video_dataset.py` 读取路径。

### 2.2 训练/评测泄漏去重（不做会被审稿人抓死）

- 威胁模型：UCF-101 与 DAVIS 源自 YouTube，OpenVid/VideoUFO 也抓自 YouTube → 训练集可能包含评测视频；
- 流程：① 对 UCF/DAVIS 每段视频取 5 个均匀关键帧算 pHash（`imagehash` 库）；② 对训练集每段视频取中心帧 pHash；③ 汉明距离 ≤8 的训练 clip 全部剔除并记录清单 `dedup_removed.csv`；④ 剔除率与清单写进论文附录。
- 成本：单机 CPU 一晚（哈希是廉价操作）。

### 2.3 数据版本化

每个数据集处理完固化一个版本号（`openvid-256-v1` 等），实验卡（04 §4）引用版本号而非路径；重处理必须升版本，禁止原地覆盖。

## 3. 评测协议唯一实现（`uvt/eval/protocols.py`）

全仓库所有重建评测**只允许**经过这一个文件（Hydra-X 表 9 强调 identical scripts；预处理不统一是重建指标不可比的头号原因）：

```python
# 图像: 短边resize到256(bicubic, antialias=True) → center crop 256 → [0,1]→[-1,1]
# 视频: 每段取前17帧起点为第一个可用帧(DAVIS)/随机固定seed起点(UCF, seed=0固定全表);
#       逐帧同上空间处理; 输出 [3,17,256,256]
# 指标:
#   PSNR/SSIM: 逐帧计算后对帧取均值(pytorch-msssim的SSIM, data_range=2.0)
#   rFID:  ImageNet val 50k 重建 vs 真实, pytorch-fid Inception-v3 pool3
#   rFVD:  I3D logits版(styleganv实现, 主选)——16帧窗口, 取clip前16帧;
#          videogpt版并行输出仅作交叉核对, 论文只报主选
#   LPIPS: vendored AlexNet版
```
- rFVD 实现二选一的裁决方法：P0 校准中哪个实现能同时复现 Wan2.2 与 OmniTokenizer 的公开数字排序与量级，即定为主选并写死；
- 17 帧 clip vs I3D 16 帧窗口的处理（取前 16 帧）必须写进论文评测细节。

## 4. P0 校准（五锚点，Gate 见 04 §2-P0）

流程（每个锚点重复）：
1. **官方权重 + 官方脚本** → 复现其 README/论文数字（验证环境与权重完好）；
2. **官方权重 + 我们的 protocols.py** → 得到"我们尺子下"的数字；
3. 记录两套数字 → 产出《校准报告》：五家在我们尺子下的**排序**必须与公开数字排序一致；绝对值偏差记录在案（预期 PSNR ±0.1dB 内、rFID/rFVD ±10~20% 属正常实现差异）。

校准完成的副产品：表 9 型对比表的基线行（AToken/Wan2.2/OmniTokenizer/LARP/LeanVAE）直接生成，论文对比全部出自同一把尺子。

## 5. 三线评测详细协议

### 5.1 重建线（证明"保真"，零额外训练）

- ImageNet val 50k：PSNR/SSIM/rFID；DAVIS：PSNR/SSIM/rFVD；UCF-101 val：同 DAVIS；
- 每个 Gate 快照评测用 DAVIS（小，快）；正式对比表三个集全跑。

### 5.2 语义线（证明"有语义"，几乎零训练）

| 探针 | 协议 | 成本 |
|---|---|---|
| **zero-shot ImageNet** | 蒸馏对齐后，s_pool 经 SigLIP2 逻辑（其 prompt 模板 + 文本塔嵌入）算相似度分类；报 top-1 | 零训练（ADR-9 的直接回报） |
| **linear probe（图像）** | 冻结 tokenizer，锚帧 s 均值池化 → 单线性层，ImageNet 90 epoch，lr 扫 {1e-3,3e-3,1e-2}×bs512 | ~50 GPU·h |
| **linear probe（视频）** | K400 与 SSv2：clip 的 s[:,1:] 时空均值池化 → 线性层；SSv2 是时间敏感主判据 | 各 ~80 GPU·h |
| **CKNNA** | 与 **DINOv2-L**（第三方参照，与两教师独立）在 ImageNet val 10k 上算逐层对齐曲线；**禁止**以教师自身为参照（循环论证） | 忽略不计 |

### 5.3 可生成性线（证明"latent 好生成"，标准做法）

- **图像**：OmniTokenizer 仓库 `Diffusion/DiT`（docker 外可独立跑，其 DiT 代码无 PL 依赖）：DiT-B/1 结构（latent 网格 16×16 无需再 patchify），`in_channels=64`（我们的 ẑ，ADR-10 规范化后），ImageNet 256 类条件，400k 步 bs 256，ADM 套件报 **gFID-50k**（cfg=1.5 与 no-cfg 双报）；
- **对照组**（同 DiT 配置同步数同数据，唯一变量是 latent）：OmniTokenizer-VAE latent（C=8）、SD-VAE latent（C=4, 32×32）；报告须注明通道数/网格差异这一混杂（README D12 注意事项）；
- **视频**：`Diffusion/Latte` UCF-101 类条件（其 `ucf101_train_omnitokenizer.yaml` 改 latent 接口即用），对照 LeanVAE latent，报 FVD；
- 判读规则：报 gFID 随训练步数的**曲线**而非单点（收敛速度本身是"可生成性"的一部分）。

### 5.4 表征级 STI 评测（P5 第二实验）

- 数据：ImgEdit val 的 1k 对；两组模型（STI 路由 vs Indep 路由，同 checkpoint 同权重，只是推理路由不同——注意这是**推理期路由对比**，因为 Sem-ViT 的 tubelet mask 天然支持两种喂法；若发现需要训练期适配，则从 P4 checkpoint 各微调 10k 步，配对 seed）；
- 指标①（sanity）：decoder(z_src) 的 PSNR 两组必须相同（z 未经 Sem-ViT，应严格一致）；
- 指标②（主判据）：用 s_tgt 线性回归源图的 16× 下采样像素（一个 ridge 回归探针，闭式解），STI 组的重建误差应显著低于 Indep 组——直接度量"目标表征吸收了多少源图结构"；
- 指标③：s_src 在两种路由下的一致性（cosine>0.999，验证因果 mask 无泄漏）。

## 6. 评测排期与自动化

- 每个训练 run 结束自动触发：DAVIS 快照评测 + 实验卡回填（04 §4）；
- 每个 Phase Gate 前手动触发全量三线评测；
- 所有评测任务单卡可跑（DiT 训练除外），排队在训练空隙。
