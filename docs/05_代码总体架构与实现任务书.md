## §0 全局约定（所有任务卡的公共前提，违反即打回）

1. **张量约定**：token 张量统一 `[B, T1, N, D]`（T1=1+折叠后时间位数，位 0=锚帧；N=每帧空间 token 数；纯图像 T1=1）。进注意力时展平为 `[B, T1*N, D]`，时间位内空间 token 连续排布。像素张量 `[B, 3, F, H, W]`（图像允许 `[B,3,H,W]`，入口处升维）。latent `[B, T1z, N, 64]`。
2. **接口冻结**：任务卡里写明的类名/函数签名/形状是契约，实现者不得更改；发现契约有问题→提出来改任务书，不许私改代码。
3. **禁止事项**：不引入 pytorch-lightning；不引入 flex_attention 依赖（mask 用 SDPA additive bias）；歧义处一律做成 config 字段并引用 ADR 编号，禁止硬编码。
4. **测试先行**：每张卡的"验收"栏就是要先写的测试；单测全绿 + `csv_file=null128` 假数据端到端冒烟通过 = 完成标准。
5. **配置对象**：全局 `UVTConfig`（dataclass，卡 M-10 定义）贯穿所有模块，字段含默认值与 ADR 出处。
6. 环境：torch 2.4 + cu121 + transformers≥4.47（主仓）；Phase B 独立 docker（见 02 篇 §1.2）。

### M-2 `models/temporal_fold.py` ✅ 已交付
职责：`TemporalFold2x`/`TemporalUnfold2x`，锚帧永不折叠（ADR-4'），近恒等初始化保护预训练分布。验收：`test_fold_shapes`（(1+16)→(1+4)）、`test_image_passthrough`。

### M-3 `models/blocks.py` ★关键卡（新写，~200 行）
- **职责**：与 SigLIP2 权重兼容的 transformer block，支持自定义 mask 与时间 RoPE——这是全模型的原子构件。
- **接口（冻结）**：
```python
class UVTAttention(nn.Module):
    def __init__(self, dim: int, num_heads: int, rope_dims: int = 32): ...
    def forward(self, x: Tensor,              # [B, S, D]
                attn_bias: Optional[Tensor],   # [1,1,S,S] 或 None（来自 M-1）
                time_ids: Optional[Tensor]     # [S]，None=不加时间RoPE
                ) -> Tensor: ...               # [B, S, D]

class UVTBlock(nn.Module):                     # pre-LN: ln1→attn→残差, ln2→mlp→残差
    def forward(self, x, attn_bias, time_ids) -> Tensor: ...
    @classmethod
    def from_siglip(cls, layer) -> "UVTBlock": ...   # 从 HF SiglipEncoderLayer 深拷贝权重
```
- **实现要点**：① q/k/v/out 四个 Linear 从 `layer.self_attn.{q,k,v,out}_proj` 深拷贝，ln1/ln2/mlp 从 `layer.{layer_norm1,layer_norm2,mlp}` 深拷贝——**不要直接调用 HF layer.forward**（其 mask 语义随版本漂移，自持权重才可控）；② 时间 RoPE（ADR-8）：仅对每头前 `rope_dims`（偶数，默认 32，head_dim=72）做 1D rotary，角度=time_id×频率（10000 分频），rotate-half 惯例；`time_ids=None` 或 `rope_dims=0` 时跳过（消融臂开关）；③ 注意力用 `F.scaled_dot_product_attention(q,k,v,attn_mask=attn_bias)`。
- **验收**：`test_block_weight_transfer`——tiny SigLIP 层与 UVTBlock 在 time_ids=None、bias=None 下对同一输入输出 allclose(atol=1e-5)；`test_rope_shift_equivariance`——整体平移 time_ids 不改变注意力分数差异模式。

### M-4 `models/siglip_backbone.py`（新写，~100 行）
- **职责**：加载/切分 SigLIP2，提供 tiny 离线模式（测试用）。
- **接口（冻结）**：
```python
@dataclass
class BackboneParts:
    embeddings: nn.Module        # SiglipVisionEmbeddings 深拷贝（patchify+学习式空间PE, ADR-8）
    gen_blocks: list[UVTBlock]   # 第 1..gen_depth 层
    sem_blocks: list[UVTBlock]   # 其余层
    dec_blocks: list[UVTBlock]   # 全部 27 层的另一份深拷贝（decoder 初始化, 01 §2.7）
    post_ln: nn.Module
    map_head: nn.Module          # MAP 注意力池化头深拷贝（ADR-9 的 zero-shot 关键）
def load_siglip_parts(model_name="google/siglip2-so400m-patch16-256",
                      gen_depth=13, tiny=False) -> BackboneParts: ...
```
- **实现要点**：① 用 `SiglipVisionModel.from_pretrained(...).vision_model`，断言 27 层/1152 宽；② **用固定分辨率 256 权重而非 naflex**（网格与训练分辨率 1:1 对齐，naflex 的变分辨率打包接口徒增复杂度；此为对 Hydra-X 的实现级偏离，P4 混合分辨率阶段再评估 naflex）；③ `tiny=True` 时用 `SiglipVisionConfig(hidden_size=64, num_hidden_layers=6, num_attention_heads=2, image_size=64, patch_size=16)` 随机初始化——所有单测离线可跑。
- **验收**：`test_load_split`（层数分配 13/14/27）、`test_tiny_offline`（无网络环境构建成功）。

### M-5 `models/gsb.py`（新写，~60 行）
- **职责**：连续瓶颈（压缩/重参数化/KL/规范化接口）。
- **接口（冻结）**：01 篇 §2.6 的类签名原样（`compress/to_canonical/from_canonical`），**删去其中的 `expand`**——反投影按 HYDRA 的 `W_unproj^und` 上标裁决拆到消费方：Sem-ViT 与 decoder 各自持有独立 `in_proj: Linear(64→D)`（否则 Stage-3"冻 decoder 训 Sem 侧"会在共享投影上打架）。
- **验收**：`test_gsb_roundtrip`（规范化往返）、`test_kl_zero_at_standard_normal`（μ=0,ρ=0 时 KL≈0）。

### M-6 `models/encoder.py`（新写，~120 行）
- **职责**：GenViT——逐帧 patchify → 折叠位循环 → gen blocks。
- **接口（冻结）**：
```python
class GenViT(nn.Module):
    def __init__(self, parts: BackboneParts, cfg: UVTConfig): ...
    def forward(self, video: Tensor,           # [B,3,H,W] 或 [B,3,1+T,H,W], (T)%4==0
                attn_mode: str = "tubelet") -> Tensor:  # h: [B, 1+T/4, N, D]
```
- **实现要点**：① 图像入口升维 `[B,3,1,H,W]`；② 逐帧过 `embeddings`（(B·F) 折进 batch）；③ 折叠位置 `cfg.fold_positions=(0,6)`（ADR-3 默认，config 可换）：位置 i 表示"在第 i 个 block 之前折叠"，位置 0=输入处；④ 每次 T1 变化后重取 `attn_bias` 与 `time_ids`（M-1 有缓存，直接调用）；⑤ 折叠次数固定 2（4× 时间压缩）。
- **验收**：`test_encoder_shapes`（17 帧 →1+4 位；纯图像 →1 位）、`test_fold_position_config`（(0,4) 配置下第 4 块前折叠生效——用 hook 检查各块输入形状）。

### M-7 `models/sem_vit.py`（新写，~120 行）
- **职责**：语义读出支路 + MAP 池化 + STI 双路由。
- **接口（冻结）**：
```python
class SemViT(nn.Module):
    def __init__(self, parts: BackboneParts, cfg: UVTConfig): ...   # 含 in_proj: Linear(64→D)
    def forward(self, z_can: Tensor,           # [B,T1,N,64] 规范化latent
                attn_mode: str = "tubelet"
                ) -> tuple[Tensor, Tensor]:    # s:[B,T1,N,D], s_pool:[B,D]（锚帧token经MAP头）
    def forward_pair(self, z_src: Tensor, z_tgt: Tensor  # 各[B,1,N,64]
                ) -> tuple[Tensor, Tensor]:    # s_src, s_tgt —— STI: 拼成T1=2, tubelet mask
```
- **实现要点**：① `s_pool` 只对锚帧 token（`s[:,0]`）过 map_head——图像级语义接口，zero-shot 用它；② STI 的因果保证由 tubelet mask 天然给出（位 0=源只见自己）；对照组 Indep 就是对两图分别调 `forward`，**不要单写 Indep 分支**（评测脚本负责路由）。
- **验收**：`test_sti_causality`——`forward_pair` 输出的 s_src 与单独 `forward(z_src)` 的 s[:,0] allclose（源侧无泄漏，03 篇 STI 指标③的单测版）。

### M-8 `models/decoder.py`（新写，~130 行）
- **职责**：像素解码器（27 层 + 镜像展开 + 双像素头）。
- **接口（冻结）**：
```python
class PixelDecoder(nn.Module):
    def __init__(self, parts: BackboneParts, cfg: UVTConfig): ...  # 含 in_proj: Linear(64→D)
    def forward(self, z_phys: Tensor,          # [B,T1z,N,64] 物理latent(非规范化)
                hw: tuple[int, int],           # 目标(H,W), N 必须等于 (H/16)*(W/16)
                attn_mode: str = "tubelet") -> Tensor:  # x̂: [B,3,1+T,H,W]
```
- **实现要点**：① 展开位置 `cfg.unfold_positions=(21, 27)`（27=末块之后），镜像 encoder 的 (0,6)；② 锚帧/非锚帧独立像素头 `Linear(D→3·16·16)`（镜像 OmniTokenizer 的 first-frame 分头设计）；③ dec_blocks 用 `parts.dec_blocks`（SigLIP2 权重复制初始化，01 §2.7；对照臂随机初始化由 config `decoder_init` 控制）。
- **验收**：`test_decoder_shapes`（latent 1+4 位 → 像素 17 帧）、`test_image_roundtrip_shape`（图像→z→图像形状闭环）。

### M-9 `models/decompressor.py`（新写，~70 行）
- **职责**：训练期 4× 时间上采样（蒸馏专用，checkpoint 不导出）。
- **接口（冻结）**：`Decompressor(dim, teacher_dim, num_heads).forward(s_frames: [B,T1z-1,N,D]) -> [B,4*(T1z-1),N,teacher_dim]`（输入不含锚位）。
- **实现要点**：两级 `(TemporalUnfold2x → UVTBlock(full mask, 无RoPE或时间RoPE皆可, 定full))` + 末端 Linear 到教师维度。**注意**：输入没有锚位，TemporalUnfold2x 的锚帧直通逻辑会误伤——调用前补一个哑锚位或在本模块内直接用 `proj+rearrange`（实现者二选一，写明选择）。
- **验收**：`test_decompressor_shape`（4 位→16 帧，与教师逐帧对齐）。

### M-10 `models/uvt_tokenizer.py` ★总装卡（新写，~180 行）
- **职责**：`UVTConfig` 定义 + 模型总装 + 训练前向 + 阶段冻结 + HF 发布接口。
- **接口（冻结）**：
```python
@dataclass
class UVTConfig:
    model_name: str = "google/siglip2-so400m-patch16-256"
    gen_depth: int = 13                 # ADR: HYDRA 均衡切分
    c_latent: int = 64                  # HYDRA C=64 甜点
    fold_positions: tuple = (0, 6)      # ADR-3, P2-pre 后冻结
    unfold_positions: tuple = (21, 27)
    attn_mode: str = "tubelet"          # {full,causal,tubelet}, P2 消融
    rope_dims: int = 32                 # ADR-8, 0=关(消融臂)
    decoder_init: str = "siglip"        # {siglip,random}, P1 消融
    use_cos_consistency: bool = True    # HYDRA L_cos, P1 消融
    kl_weight: float = 1e-6             # D14
    tiny: bool = False

class UVTTokenizer(nn.Module, PyTorchModelHubMixin):   # 发布方式学 LARP
    def __init__(self, cfg: UVTConfig): ...            # 组装 GenViT/GSB/SemViT/Decoder/Decompressor
    def encode(self, video, sample=True) -> dict:      # {z, mu, kl, h}  z为物理latent
    def decode(self, z_phys, hw) -> Tensor
    def forward_train(self, video) -> dict:            # {x_hat, z, mu, kl, h, s, s_pool, decomp_out}
    def semantic(self, video) -> tuple[Tensor,Tensor]  # 推理用: (s, s_pool), 内部走规范化z
    @torch.no_grad()
    def estimate_latent_stats(self, loader, n=10_000)  # Stage3: 统计z通道mean/std写入GSB并置normalize
    def set_stage(self, stage: int)                    # 1/2/3 冻结策略(01 §2.9 表)
```
- **实现要点**：`forward_train` 里 Decompressor 仅在输入为视频时调用；`state_dict` 钩子里剔除 decompressor（训完丢弃）；`set_stage(2)` 冻 encoder+GSB、`set_stage(3)` 只留 SemViT+map_head 可训。
- **验收**：`test_forward_tiny`（tiny 配置，视频/图像各一遍，所有输出形状与有限性断言）、`test_stage_freeze`（各 stage 的 requires_grad 计数断言）、02 篇 `test_tokenizer_overfit_single_batch`。

### T-2 `teachers/internvideo_teacher.py`（新写，~80 行）
- 职责：冻结视频教师 + 协议适配。接口：`forward(video_01: [B,3,16,H,W]) -> feats[B,16,N_t,D_t]`（丢弃 CLS/聚合 token，ADR-7）。config 字段 `teacher_id` 支持 InternVideo-Next-L 与备胎 InternVideo2-Stage2-1B 切换；空间网格与我方不一致时由 L-2 做双线性对齐（教师侧只管原样输出）。
- 验收：golden fixture（P0-golden 首日生成入库）；**该卡有真实的 API 不确定性**（InternVideo-Next 加载方式需读其官方 repo），实现者需先做 30 分钟接口勘察再动工。

### L-1 `losses/recon.py`（新写，~60 行）
- 接口：`recon_loss(x, out: dict, cfg) -> dict{l1, lpips, kl, cos_consistency, total}`。LPIPS 用 vendored 权重；L_cos = 1−cos(SemViT.in_proj(μ_can) 的 D 维向量, h)（HYDRA 式 4；实现按 token 平均）。
- 验收：`test_recon_loss_perfect`（x_hat==x 时 l1/lpips≈0）。

### L-2 `losses/distill.py`（新写，~80 行）
- 接口：`distill_loss(s, s_pool, decomp_out, t_img_patch, t_img_pool, t_vid, is_video: Tensor[B]) -> dict{img_patch, img_pool, vid, total}`。三项 cosine 距离（01 §2.8）；教师/学生维度不一致处：patch 项由 SemViT 侧 head 早已对齐（D 一致则免）——**裁决：加一个可学习 Linear 头 per 项**（学生→教师维度），头参数属于训练期附件（同 Decompressor 不导出）；纯图像 batch 的 vid 项返回 0 且不参与反传；空间网格不一致时对 decomp_out 做 `F.interpolate` 双线性对齐（ADR-7）。
- 验收：`test_distill_masking`（is_video 全 False 时 vid 项梯度为 None）。

### L-3 `losses/gan.py`（**LARP 移植**，~改 60 行）
- 来源：`LARP/models/loss.py` 的 `lpips_disc_loss` 中判别器相关部分（transformer 判别器、ns_smooth、LeCam、d_update_freq、r1 可选）。
- 移植改动：① 判别器 patchify 参数改为我方协议（`disc_tran_temporal_patch_size=4, disc_tran_patch_size=16, input 17×256²`）；② 剥离其 VQ 相关损失项；③ 保持 Adam(0.5,0.9) 与 `dis_lr_multiplier` 语义。仅 Stage 2 启用。
- 验收：`test_disc_forward`（真假样本 logits 形状/有限性）、G/D 交替 100 步 loss 不 NaN（tiny 配置）。
