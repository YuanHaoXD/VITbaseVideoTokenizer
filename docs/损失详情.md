# 损失详情（UVT 全部损失函数详解）

> 配套 01 篇 §2.9 的损失总表。本篇是教学版：每个损失是什么、为什么需要、失控时什么症状。

## 0. 总模型：为什么是一堆损失而不是一个

每种损失只能"看见"一种错误。tokenizer 要同时满足"重建好、latent 规整、语义在"，没有单一损失能覆盖，所以是分工制：

```
像素输出 x̂  ←── L1（结构对不对）+ LPIPS（看起来像不像）+ GAN（纹理真不真）
latent z    ←── KL（latent 空间规整）
瓶颈两侧    ←── L_cos（信息过瓶颈不迷路）
语义特征 s  ←── 蒸馏三项（语义有没有）
```

---

## 1. 像素侧三兄弟（管重建质量）

### 1.1 L1 损失

- **定义**：重建图与原图逐像素绝对差取平均：`|x − x̂|`。
- **管什么**：大结构、颜色、位置基本正确。
- **为什么不用 L2**：L2 对大误差惩罚极重，模型会输出"所有可能纹理的平均"→ 糊。L1 更温和，出图更锐。
- **管不了什么**：不懂人类视觉。整图平移 1 像素（人眼无感）L1 报巨大误差；纹理换成同样真实的另一种（人觉得没问题）L1 也报错。**只用 L1 必然糊**——糊是逐像素意义下最安全的答案。

### 1.2 LPIPS 感知损失

- **定义**：原图与重建图分别过一个**冻结的预训练分类网络**（AlexNet/VGG），比较中间层**特征**差异。
- **为什么有效**：预训练特征天然编码人类关心的东西（边缘/纹理/部件），对微小平移不敏感。专治 L1 的糊。
- **管不了什么**：高频细节真实感——LPIPS 满意的图可能仍有蜡质、过度平滑的质感（头发丝、草叶抓不住）。

### 1.3 GAN 对抗损失

- **定义**：唯一"活的"损失。额外训练一个**判别器 D**（我们用 LARP 的 transformer 判别器）区分真图/重建图；decoder 的任务之一是骗过它。两者对抗共训：
  
  ```
  判别器 D：看图打分"真的还是重建的？"（学习找茬）
  decoder ：调整重建让 D 打不出假分     （学习消除"假的痕迹"）
  ```
- **为什么需要**：L1/LPIPS 都是"和原图比"，GAN 是"和真实图像的整体分布比"——逼重建落在"像真照片"的流形上，把蜡质感磨掉。
- **代价**：出名地不稳定（两个网络互相追逐）。我们的稳定器：
  - 只在 Stage 2 引入且 encoder 已冻结（波及面最小）；
  - **LeCam 正则**：惩罚判别器过度自信（D 太强时 decoder 收不到有用梯度）；
  - **d_update_freq=5**：D 每 5 步才更新一次；
  - **ns_smooth**：生成器侧损失变体，梯度性质更好；
  - D 用 Adam(0.5, 0.9)——GAN 社区经验值。
- **无 GAN 备胎（P4 平行臂）**：Gram 矩阵损失（AToken；特征相关性统计=纹理"统计指纹"）、DINOv3 感知损失（ViTok-v2；LPIPS 的现代版）。既是保险也是卖点。

## 2. latent 侧两个（管压缩空间质量）

### 2.1 KL 正则

- **定义**：GSB 输出分布（μ, ρ），KL 散度惩罚其偏离标准正态 N(0,1)。
- **为什么需要**：没有它 encoder 会把 latent 甩到空间任意角落、用任意大数值——重建照样好，但空间坑坑洼洼没有连续性，下游 diffusion 没法建模。KL 把 latent 往原点附近拢，保证平滑、有界。
- **为什么权重只有 1e-6**：KL 与重建天然对抗——拉得越紧信息越少、重建越糊。**KL 过大是"重建莫名发糊"的头号嫌疑**。起点 1e-6 来自 OmniTokenizer 实证值，P1 扫 {1e-6, 1e-5, 1e-4}。

### 2.2 L_cos 一致性损失（HYDRA 独有，我们做成开关）

- **定义**：压缩后的 μ 反投影回 1152 维，要求与压缩前特征 h 方向一致（cosine 对齐），λ=1.0。
- **直觉**：信息压到 64 维再展开难免变形；此损失要求走完"压缩-展开"后特征仍大致指向原方向——保证 Sem-ViT 接手的东西与 Gen-ViT 交出的东西连贯。

## 3. 语义侧蒸馏三项（管 latent 有没有语义）

- **定义**：余弦距离 `d_cos(a,b)=1−cos(a,b)`——只比方向不比长度（特征模长在不同网络间不可比，方向携带语义，蒸馏行规）。

| 项          | 学生                      | 老师                     | 管什么                                                 |
| ---------- | ----------------------- | ---------------------- | --------------------------------------------------- |
| 锚帧 patch 项 | Sem-ViT 锚帧 256 token    | SigLIP2 的 256 patch 特征 | 逐块图像语义                                              |
| 池化项        | MAP 头池化的整图向量 s_pool     | SigLIP2 整图嵌入           | 图像级语义；**继承文本塔做 zero-shot 的前提**（SigLIP2 图文对齐住在池化嵌入里） |
| 视频项        | Decompressor 升回 16 帧的特征 | InternVideo 的 16 帧特征   | 运动/时序语义（纯图像 batch 关闭）                               |

- **意义**：没有这三项，latent 只是高保真压缩包（OmniTokenizer/LeanVAE 止步于此）；有了它们才是"统一 tokenizer"——这是与普通视频 VAE 的分界线，也是论文主卖点。

## 4. 按阶段分工总表

| 阶段         | 开着的损失                          | 一句话理由                                    |
| ---------- | ------------------------------ | ---------------------------------------- |
| Stage 1 基础 | L1 + LPIPS + KL + 蒸馏三项 + L_cos | 结构、感知、空间规整、语义一起打地基；不开 GAN（前期不稳时加对抗是自找麻烦） |
| Stage 2 精修 | L1 + LPIPS + KL + **GAN**（关蒸馏） | encoder 冻结后专攻纹理真实感                       |
| Stage 3 调和 | 蒸馏三项                           | 只微调语义侧适应归一化后的 latent 接口                  |

## 5. 失控症状速查表（调参诊断手册）

| 症状                     | 头号嫌疑                                     |
| ---------------------- | ---------------------------------------- |
| 重建整体发糊                 | KL 权重过大；或只有 L1 没有 LPIPS                  |
| 蜡质感、细节平滑               | 缺 GAN 或 GAN 权重太小                         |
| 出现原图没有的纹理/伪影、损失剧烈震荡    | GAN 权重过大或判别器失衡 → 查 LeCam / d_update_freq |
| 重建好但 zero-shot/probe 差 | 蒸馏权重太小，或池化项没起作用                          |
| 语义指标好但重建掉              | 蒸馏权重太大（λ_dist 下调，P3 扫描的意义）               |
| latent 数值发散、下游 DiT 训不动 | KL 太小 + 没做 Stage 3 归一化                   |

## 6. 容易混淆但不属于我们主线的损失

- **logits-laplace**：OmniTokenizer 的 L1 变体，我们不用；
- **VQ commitment/codebook 损失**：离散路线专用，只在 Phase B 的 OmniTokenizer 原配方里遇到；
- **flow matching / 扩散损失**：Phase 5 验证时训 DiT 用的，属于下游模型，不属于 tokenizer 本体；
- **交叉熵（下一 token 预测）**：LARP 的 AR prior 用的，离散路线。
